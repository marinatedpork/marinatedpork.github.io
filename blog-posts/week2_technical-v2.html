Display inline vs inline-block

July 24, 2014

Know the values

Part of putting together a well designed page means having control over how different boxes and containers behave around each other. When we introduce new elements into our page, like a header section or a social media button, we have must establish these behaviors. One way to establish this behavior is to use the display property. What follows is an explanation of two different values of display:

  • Inline
  • Inline-block

Although this post does implement and mention the block value of the display property, it is not explained as in-depth as these other two values. To learn more about the block value, check out this site or see the sources below!

Display: Inline

According to the W3 Schools description of the display property1, inline is the default value of the display property. An element with the property/value display: inline means that the element is displayed inside the current block on the same line. Thus, creating no new lines. An inline element only creates a new line when it is forced to by other elements, the inline element's width however becomes the smallest possible2. Inline elements do not take width or height nor respect bottom or top margins and padding. A heplful part of them is that they allow elements to sit to the left and right. Notice in the example above that the display inline element is being forced onto a new line by the block elements around it. Additionally, the block elements take up as much width as they can and the inline element takes up the minimum amount of width. Finally, the inline element respects the left and right margins that we've given it, but not the top and bottom.


Here's the CSS syntax for the classes "inline" and "forcer2", which we've assigned to sections above. Can you guess which section has which class? 

.inline {
display: inline;
border: solid black;
background-color: #213C6E;
margin: 5px 20px 5px 20px;
padding: 2px;
}
				
.forcer2 {
display: block;
background-color: #213C6E;
border: 2px solid black;
margin: 5px 20px 5px 20px;
}

Display: Inline-Block

The inline-block value of the display property places an element as an inline-level block container. Thus, it is on the same line as adjacent elements. Additionally, the width and height may be set, and they respect all margins and padding. Take a look at the example below to see how they behave:

Notice that the inline-bock containers have respected the width and height that we have given them as well as the margins and padding. Meanwhile, content is sitting on the left and the right. Pretty neat, huh?


Here's the CSS syntax for the class "inline-block", which we've assigned to our inline-blocks above. See how it respects all the margin, padding, height and width properties we've given it?


.inline-block {
display: inline-block;
border: solid black;
background-color: #213C6E;
margin: 5px 10px 5px 10px;
padding: 2px;
width: 120px;
height: 120px;
}

Summary

There is a time and place for both of these display values, so it's important to know how to use them both. One user on Stack Overflow sums it up perfectly with the following rules for block, inline and inline-block elements3:

  • Inline elements:
    1. Respect left and right margins and padding, but not top and bottom.
    2. Cannot have a width and height set.
    3. Allow other elements to sit to their left and right.
  • Block elements:
    1. Respect all of those.
    2. Force a line break after the block element.
  • Inline-block elements:
    1. Allow other elements to sit to their left and right.
    2. Respect top and bottom margins and padding.
    3. Respect height and width.
