Responsive Design and Fluid Layouts

An exploration into dynamicisms

By: Matthew Meyers

Responsive Design


Responsive web design is the implementation of pixel breakpoints on stylesheets to maximize usability. Pixel breakpoints are boundaries you set that, once crossed by a dimensions of the user's display, initialize different style rules. For example, a designer could increase font size at a width of 320px to make content easier to read for users with a smart phone.



Design is about asking the right questions.



Responsive web design has become somewhat of an industry standard with the proliferation of tablets, smart phone, and other devices that break from the early 2000s convention of computer displays. The need for responsive design arose from the fact that most web layouts for the same on every platform. Thus, it's become the challenge of web designers to consider these changes.



Fluid Layouts

Fluid layouts are web layouts that have implemented different style rules at pixel breakpoints established in a websites stylesheet. If you view a fluid layout on a laptop or desktop browser, you can resize the browser window and watch the layout change as your window crosses certain maximum and minimum widths (i.e. breakpoints). This dynamic effect is essentially why we call them fluid layouts.

For understanding these pixel breakpoints I used the Window Resizer1 plug-in for Chrome. It was quite helpful for establishing important breakpoints for my layout and resizing my window according to device specific widths and heights. In addition, the code is pretty simple to deifine a pixel breakpoint in your style sheet:

@media (max-width: value px) { element: value }

By using the above code and estlabishing a value for "max-width", you initialize a style change for all @media with a width less than or equal to your established max-width2 3.

On this website, the top two sections have a max-width set at 1200px. Below 1201px the second secion falls below the first section. At 1025px the fox size increases to 18px and then to 20px at 640px. The goal here was to make the content easy to read as you decrease the window size.

The purpose of fluid layouts is to maximize usability. As in the above sec, text that gets bigger as the display gets smaller is easier to read. Not only does is it easier to read, it also looks better. If you want to guarrentee the best experience for the most users, it's a good idea to implement this sort fluid layouts. There are so many different devices nowadays, and you don't want scare away visitors with outdated layouts that are harder to use.
